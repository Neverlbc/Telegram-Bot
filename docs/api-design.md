# Bot 接口设计文档

## 1. Bot 命令列表

| 命令 | 说明 | 可用范围 |
|------|------|---------|
| `/start` | 启动 Bot，进入语言选择（新用户）或主菜单（老用户） | 所有用户 |
| `/menu` | 返回主菜单 | 所有用户 |
| `/help` | 显示帮助信息 | 所有用户 |
| `/lang` | 切换语言 | 所有用户 |
| `/track` | 快捷物流查询（直接输入跟踪号） | 所有用户 |
| `/support` | 联系客服 | 所有用户 |
| `/cancel` | 取消当前操作，清除 FSM 状态 | 所有用户 |
| `/admin` | 管理后台入口 | 仅管理员 |
| `/stats` | 查看统计数据 | 仅管理员 |

## 2. CallbackData 路由设计

### 2.1 命名规范

所有 callback_data 使用 aiogram 的 `CallbackData` 工厂类，格式为：

```
<模块>:<动作>:<参数1>:<参数2>:...
```

### 2.2 CallbackData 定义

```python
# 语言选择
class LangCallback(CallbackData, prefix="lang"):
    code: str  # zh / en / ru

# 主菜单
class MenuCallback(CallbackData, prefix="menu"):
    action: str  # presale / order / aftersale / device / support / settings

# 商品目录
class CatalogCallback(CallbackData, prefix="catalog"):
    action: str       # list / detail / page
    category_id: int  # 分类 ID（0 表示根级）
    page: int = 1     # 页码

# 商品操作
class ProductCallback(CallbackData, prefix="product"):
    action: str      # view / order
    product_id: int

# 售前咨询
class PresaleCallback(CallbackData, prefix="presale"):
    action: str        # menu / catalog / delivery / faq / category / product / variant / faq_detail
    category_id: int = 0
    product_id: int = 0
    variant: str = ""  # industrial / opensource / special
    faq_id: int = 0
    page: int = 1

# 售中下单
class OrderCallback(CallbackData, prefix="order"):
    action: str  # wholesale / confirm / cancel

# 售后支持
class AftersaleCallback(CallbackData, prefix="aftersale"):
    action: str       # query / status
    order_id: str = ""

# 物流查询
class LogisticsCallback(CallbackData, prefix="logistics"):
    action: str    # origin / carrier / track
    origin: str = ""   # moscow / china
    carrier: str = ""  # cdek / rupost / cainiao / airfreight

# 设备支持
class DeviceCallback(CallbackData, prefix="device"):
    action: str        # menu / serial / issue / more / type / submit
    section: str = ""  # serial / issue / more
    issue_type: str = ""  # firmware / hardware / software / remote

# 客服
class SupportCallback(CallbackData, prefix="support"):
    action: str  # business / wholesale / human / back

# 导航
class NavCallback(CallbackData, prefix="nav"):
    action: str  # back / home
    target: str = ""  # 返回目标
```

## 5. 消息模板规范

### 5.1 模板命名

翻译 key 命名规则：`<模块>.<场景>.<元素>`

```
start.welcome          — 欢迎语
start.select_lang      — 选择语言提示
menu.title             — 主菜单标题
catalog.categories     — 分类列表标题
catalog.product_detail — 商品详情模板
order.wholesale_prompt — 批发留言提示
order.transferred      — 已转接客服提示
aftersale.prompt       — 输入订单号提示
aftersale.shipped      — 已发货状态模板
aftersale.not_shipped  — 未发货 AI 回复模板
aftersale.escalated    — 自动转人工提示
logistics.select_origin — 选择发货地提示
logistics.select_carrier — 选择物流商提示
logistics.input_tracking — 输入跟踪号提示
logistics.result       — 物流轨迹模板
serial.prompt          — 输入序列号提示
serial.result          — 查询结果模板
support.menu           — 客服合作菜单
support.transfer       — 转接人工提示
error.invalid_input    — 输入错误提示
error.api_failure      — API 查询失败提示
error.rate_limit       — 频率限制提示
```

### 5.2 格式规范

- 消息解析模式统一使用 `ParseMode.HTML`
- 金额保留 2 位小数
- 物流跟踪号使用代码块格式（`<code>XXX</code>`，方便复制）
- 时间显示格式：`YYYY-MM-DD HH:mm`
- 物流轨迹按时间倒序排列（最新在上）
